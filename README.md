# Record Wall

一个零依赖的亚克力专辑墙原型，使用原生 HTML、CSS、JavaScript 和 Node.js 实现。

在目录内启动自带的本地服务：

```bash
npm start
```

浏览器访问 `http://localhost:8000`。

> 直接打开 `index.html` 仍然可以查看墙面，但在线歌单导入依赖同源接口，需要通过 `npm start`（或 `node server.js`）启动。

## 调整内容

页面把当前浏览器视口作为固定大小的画布，不会通过滚动扩展画布。每张专辑都可以用鼠标或触控拖动，并始终被约束在屏幕边界内；移动或拖动任意一个角缩放时，只要唱片的任一边缘或中心线接近其他专辑、画布的对应线，就会显示参考线并分别在水平、垂直方向自动吸附。键盘聚焦专辑后可以用方向键微调位置，聚焦任一缩放手柄后可用方向键调整大小（按住 Shift 时步长更大）。“重置布局”可以恢复默认的 `4×2` 排列和统一大小。

页面当前默认使用 `4×2` 排列，其他排列入口暂时隐藏。点击“导入歌单”后可选择网易云音乐或 QQ 音乐（默认网易云音乐），粘贴公开歌单链接或歌单 ID，程序会从不重复的专辑封面中随机抽取 8 张并渲染到墙面。网易云会先打乱完整的 `trackIds`，按布局数量的两倍抽取歌曲（默认 16 首），获取详情并去重；重复导入会重新抽取。如果这批歌曲的不重复封面数量不足，页面会直接提示，不会继续扩大抽样或用重复图片补位。

歌单导入完成后会出现“分享链接”按钮。分享状态以 Base64 URL 编码写入 `/share#share=...` 的哈希片段，包含当前布局、专辑位置和大小、曲目信息及专辑图片 URL；打开分享链接时不再请求歌单解析接口，浏览器会直接恢复墙面排布并从对应平台的图片 CDN 加载封面。

“分享图片”会在浏览器中生成一张 `1600×1000` 的 PNG 墙面图并弹窗预览，手机端可以长按图片保存，电脑端可以右键保存。生成过程通过受限的同源图片代理读取网易云或 QQ 音乐封面，避免跨域 Canvas 无法导出的问题。

`server.js` 负责安全解析网易云与 QQ 音乐链接并代理歌单数据，避免浏览器跨域限制。在 `script.js` 的 `records` 数组中可以修改默认唱片内容；CSS 中的 `.plate-*`、`.plate-reflection` 与 `.screw` 分别负责亚克力结构、环境反射和金属固定钉。

每次歌单解析完成后，服务端会向 `logs/playlist-imports.jsonl` 追加一条 JSON Lines 日志。日志包含时间、歌单 ID/名称/原始链接、音乐平台和最终执行结果；成功时记录请求、可用及返回的封面数量，失败时记录错误信息。`/share`、封面代理和静态页面请求不会写入这份日志，日志目录也不能通过网页访问。可通过 `RECORD_WALL_IMPORT_LOG` 环境变量修改日志文件位置。

日志看板仅通过 `/manage` 地址直接访问，主页不提供入口。看板按时间倒序分页展示解析结果；服务端从 JSONL 文件尾部按固定大小的数据块反向扫描，只保留当前页所需记录，不会把整个日志文件读入内存。
