# Record Wall

一个零依赖的亚克力专辑墙原型，使用原生 HTML、CSS、JavaScript 和 Node.js 实现。

在目录内启动自带的本地服务：

```bash
npm start
```

浏览器访问 `http://localhost:8000`。

> 直接打开 `index.html` 仍然可以查看墙面，但在线歌单导入依赖同源接口，需要通过 `npm start`（或 `node server.js`）启动。

## 调整内容

页面当前默认使用 `4×2` 排列，其他排列入口暂时隐藏。点击“导入歌单”后可选择网易云音乐或 QQ 音乐（默认网易云音乐），粘贴公开歌单链接或歌单 ID，程序会随机抽取 8 首歌曲，并把专辑封面渲染到墙面。重复导入会重新随机抽取。

歌单导入完成后会出现“复制分享”按钮。分享状态以 Base64 URL 编码写入 `/share#share=...` 的哈希片段，包含当前布局、曲目信息和专辑图片 URL；打开分享链接时不再请求歌单解析接口，浏览器会直接恢复墙面并从对应平台的图片 CDN 加载封面。

“分享图片”会在浏览器中生成一张 `1600×1000` 的 PNG 墙面图并弹窗预览，手机端可以长按图片保存，电脑端可以右键保存。生成过程通过受限的同源图片代理读取网易云或 QQ 音乐封面，避免跨域 Canvas 无法导出的问题。

`server.js` 负责安全解析网易云与 QQ 音乐链接并代理歌单数据，避免浏览器跨域限制。在 `script.js` 的 `records` 数组中可以修改默认唱片内容；CSS 中的 `.plate-*`、`.plate-reflection` 与 `.screw` 分别负责亚克力结构、环境反射和金属固定钉。
