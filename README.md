# Record Wall

一个支持独立夹板材质的专辑墙原型。界面与交互使用原生 HTML、CSS 和 JavaScript，亚克力与玻璃夹板由 TypeScript、Three.js 与 Vite 渲染，接口与静态资源由 Node.js 提供。

首次运行先安装依赖：

```bash
npm install
```

开发模式会在同一个地址启动 Node API 与 Vite：

```bash
npm run dev
```

生产模式会先构建 `dist`，再启动静态页面和 API：

```bash
npm start
```

浏览器访问 `http://localhost:8000`。

> Three.js 模块和在线歌单导入都需要通过本地服务访问，不能再直接双击 `index.html`。

## 夹板渲染

项目只保留 Three.js 渲染，并提供亚克力夹板与玻璃夹板两种独立材质。亚克力使用更厚、偏暖、柔和的边缘和反射；玻璃使用更薄、偏冷、锐利的切边和高光。两者都采用双层透明板几何，并配有螺旋牙纹金属固定钉、墙面投影和带凹凸细节的灰泥墙材质。顶部可以配置 0–6 盏小型单灯泡射灯，每盏灯都能沿墙顶独立拖动或用方向键微调，并保留最小安装间距；灯具与墙面光池独立定位，同时共享一个低成本顶部面光源来影响透明板与金属反射，避免灯数增加时重复执行昂贵的物理材质光照。左上角的墙面透视控制可以在左侧近景、正视和右侧近景之间连续调整，并通过透视相机形成真实的近大远小关系。

分享图片由当前 Three.js 墙面场景离屏渲染，不再维护一套容易失真的 Canvas 2D 仿制逻辑。

## 调整内容

页面把当前浏览器视口作为固定大小的画布，不会通过滚动扩展画布。每张专辑都可以用鼠标或触控拖动，并始终被约束在屏幕边界内；移动或拖动任意一个角缩放时，只要唱片的任一边缘、中心线或相邻模块间距接近其他专辑的对应关系，就会显示参考线并分别在水平、垂直方向自动吸附。把唱片放到两个相邻模块之间时，也可以自动形成左右或上下等间距。点击任意唱片会从右侧打开独立配置抽屉，可以为每张专辑分别选择亚克力或玻璃夹板。键盘聚焦专辑后可以用方向键微调位置，按 Enter 或空格打开配置，聚焦任一缩放手柄后可用方向键调整大小（按住 Shift 时步长更大）。“重置布局”可以恢复默认的 `4×2` 排列、统一大小和亚克力夹板。

页面当前默认使用 `4×2` 排列，其他排列入口暂时隐藏。点击“导入歌单”后可选择网易云音乐或 QQ 音乐（默认网易云音乐），粘贴公开歌单链接或歌单 ID，程序会从不重复的专辑封面中随机抽取 8 张并渲染到墙面。网易云会先打乱完整的 `trackIds`，按布局数量的两倍抽取歌曲（默认 16 首），获取详情并去重；重复导入会重新抽取。如果这批歌曲的不重复封面数量不足，页面会直接提示，不会继续扩大抽样或用重复图片补位。

歌单导入完成后会出现“分享链接”按钮。分享状态以 Base64 URL 编码写入 `/share#share=...` 的哈希片段，包含当前布局、专辑位置、大小和夹板类型、射灯数量与位置、曲目信息及专辑图片 URL；打开分享链接时不再请求歌单解析接口，浏览器会直接恢复墙面排布并从对应平台的图片 CDN 加载封面。

“分享图片”会直接导出当前 Three.js WebGL 画布（包含透视、玻璃/亚克力材质、金属螺丝、射灯与反射），沿用页面当前的相机位置、纵横比和实际绘制像素，因此唱片尺寸、位置与墙面留白会和原画面逐项对齐，同时确保 AgX tone mapping、曝光和 sRGB 色彩一致。手机端可以长按图片保存，电脑端可以右键保存；封面仍通过受限的同源图片代理读取，避免跨域导出失败。

`server.js` 负责安全解析网易云与 QQ 音乐链接并代理歌单数据，避免浏览器跨域限制。在 `script.js` 的 `records` 数组中可以修改默认唱片内容；`src/three/acrylic-comparison.ts` 负责生成亚克力、玻璃、金属固定件的几何、材质与灯光。

每次歌单解析完成后，服务端会向 `logs/playlist-imports.jsonl` 追加一条 JSON Lines 日志。日志包含时间、歌单 ID/名称/原始链接、音乐平台和最终执行结果；成功时记录请求、可用及返回的封面数量，失败时记录错误信息。`/share`、封面代理和静态页面请求不会写入这份日志，日志目录也不能通过网页访问。可通过 `RECORD_WALL_IMPORT_LOG` 环境变量修改日志文件位置。

日志看板仅通过 `/manage` 地址直接访问，主页不提供入口。看板按时间倒序分页展示解析结果；服务端从 JSONL 文件尾部按固定大小的数据块反向扫描，只保留当前页所需记录，不会把整个日志文件读入内存。
